# Polymorphic Memory Resources

std::pmr — это пространство имён Polymorphic Memory Resources (полиморфные ресурсы памяти). Они появились в C++17 и определены в заголовочном файле `<memory_resource>`.

pmr даёт возможность менять стратегию выделения памяти во время выполнения, не меняя тип контейнера.

Ключевой тип — `std::pmr::memory_resource`. Это абстрактный интерфейс, с помощью которого можно реализовать свой класс, переопределив  do_allocate, do_deallocate, do_construct и т. д. Указатель на такой ресурс можно передавать в pmr‑совместимые контейнеры.

Зачем это нужно? Это эффективный механизм оптимизации приложения. Например, может быть реализованы Memory arenas/regions. Основная идея состоит в том, что приложение выделяет большой объём памяти (arena) средствами операционной системы, а отдельные объекты создаются внутри arena. Если одна arena связана только с одним потоком исполнения, то не нужны _thread safe_ механизмы, что устраняет одну из главных причин не эффективного управления ресурсами. Также в этом случае, не резко снижается количество вызовов подсистемы выделения памяти операционной системы и ещё резко снижается фрагментация "внешней" памяти. Освобождение всей arena осуществляется тогда, когда алгоритм обработки завершает свою работу.

С memory arenas связан термин **winking out** - смаргивание/моргание - все выделенные ресурсы освобождаются одним действием - завершением использования arena.

Пример использования:

```cpp
#include <vector>
#include <string>
#include <memory_resource>

int main() {
    std::pmr::monotonic_buffer_resource buffer;
    std::pmr::vector<int> v(&buffer);      // вектор использует buffer
    std::pmr::string s("hello", &buffer); // строка тоже использует buffer

    // Вся память, выделенная этими контейнерами, будет освобождена
    // при уничтожении buffer (или при вызове release/upstream)
}
```
