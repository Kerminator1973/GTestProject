# Модульная система в C++ 20

Одним из наиболее устаревших решений в C++ до С++ 20 было использование `#include` для подключение зависимостей (заголовочных файлов). Даже если мы разрабатываем приложение, которое выводит на экран сообщение "Hello, World!", это требует загузки и пре-процессинга полумиллиона строк кода. Это очень долго! Также использование директивы `#include` ведёт к нарушению **one-definition rule** (ODR).

Также важно заметить, что последовательность следования директив include имеет значение в С++. Это очень плохо, т.к. кроме всего прочего это может приводить к циклическим зависимостям.

Следует принять во внимание, что очень сложно выполнять инкапсуляцию определений, которые могут быть нужны вам и являются частью некоторого заголовочного файла.

Все эти негативные моменты привели к тому, что в C++ 20 появилась модульная система.

Импорт определений для использования теперь выглядит так:

```cpp
import my_module;
```

Пример реализации модуля. Файл "math_utils.cppm":

```cpp
export module math_utils;

import <iostream>;

// Экспортируем функцию сложения
export int add(int a, int b) {
    return a + b;
}

// Экспортируем класс Calculator
export class Calculator {
public:
    int multiply(int x, int y) {
        return x * y;
    }
};

// Не экспортируемая функция (видна только внутри модуля)
void internal_log(const std::string& message) {
    std::cout << "[LOG] " << message << std::endl;
}
```

Пояснение:

- `export module math_utils;` — объявляет модуль с именем math_utils
- `import <iostream>` — импортирует стандартный модуль для работы с потоками ввода-вывода
- `export int add(...)` — функция add будет доступна другим модулям
- `internal_log` не имеет export, поэтому она видна только внутри math_utils

>Следует заметить, что файлы cppm не поддерживались в CMake и при использовании модульной системы с CMake-проектах рекомендуется использовать расширение "cpp".

Пример использования разработанного модуля (main.cpp):

```cpp
import math_utils; // Импортируем наш модуль

#include <iostream>

int main() {
    // Используем экспортированную функцию
    int sum = add(5, 3);
    std::cout << "Sum: " << sum << std::endl;

    // Используем экспортированный класс
    Calculator calc;
    int product = calc.multiply(4, 7);
    std::cout << "Product: " << product << std::endl;

    return 0;
}
```

Пояснение:

- import math_utils; — подключает наш модуль. Не нужно писать #include "math_utils.h"
- Доступны только экспортированные сущности (add и Calculator)
