# Concepts в C++20

Concepts — это именованный набор ограничений (constraints), где каждое ограничение предписывает одно или несколько требований для одного или нескольких параметров шаблона. Концепты оцениваются на этапе компиляции: компилятор проверяет, удовлетворяют ли аргументы заданным ограничениям.

Предположим, что мы написали следующий шаблонный код:

```cpp
template<typename T>
voo foo(T& t) {...}
```

При использовании concepts можно было бы написать тоже самое вот так:

```cpp
void foo(std::swappable auto& t) {...}
```

Строка вверху означает, что тип параметра метода должен поддерживать `std::swap`.

Чаще всего программисты используют уже предопределённые concepts, которые "живут" в файле `<concepts>` и разделены по четырём категориям:

- Базовые концепции языка, такие как derived_from, integral, swappable и move_constructible
- Концепции сравнения, такие как boolean_testable
- Концепции объектов: movable, copyable, semiregular и regular
- Концепции, связанные с вызовом: invokable, predicate и strinct_weak_order

Дополнительные концепции определены также в `<iterator>` и `<ranges>`.

Практические примеры:

- `std::same_as<K, T>` — проверяет, представляют ли T и K один и тот же тип.
- `std::integral<T>` — проверяет, является ли T целочисленным типом.
- `std::floating_point<T>` — проверяет, является ли T типом с плавающей точкой.

Пример применения concepts:

```cpp
#include <iostream>
#include <concepts>

template <typename T>
concept sum_types = std::same_as<T, int> || std::same_as<T, double>;

template <sum_types T>
T sum(T a, T b) {
    return a + b;
}

int main() {
    std::cout << sum(10, 3) << std::endl;     // 13
    std::cout << sum(10.6, 3.2) << std::endl; // 13.8
}
```

Концепции прямо связаны с шаблонным кодом (templates) и их цель - сделать назначение шаблонного кода понятными, а ограничения более очевидными. Поскольку, множество ошибок при программировании на С++ связаны именно с использованием шаблонов и их сложнее всего интерпретировать, concepts - это попытка сделать такой код проще и понятнее.

Преимущества использования концептов

- Улучшение читаемости сообщений об ошибках. Раньше для ограничения шаблонов использовали SFINAE (_Substitution Failure Is Not An Error_), что приводило к громоздким и запутанным сообщениям об ошибках. Концепты делают их понятнее
- Явная спецификация требований. Концепты явно указывают, какие типы допустимы в качестве параметров шаблона, что улучшает документацию кода
- Упрощение перегрузки функций. Можно создавать разные реализации функций для разных концептов, и компилятор выберет подходящую на основе аргументов
- Сокращение времени компиляции. Компилятор может быстрее отбрасывать неподходящие шаблоны, не пытаясь их инстанцировать
- Замена сложных техник SFINAE более простым и понятным синтаксисом
