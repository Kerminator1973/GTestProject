# Советы по организации кода

Передача внешнего объёкта:

```cpp
void A(Resource*);
void B(Resource&);
void C(std::unique_ptr<Resource>);
void D(std::unique_ptr<Resource>&);
void E(std::shared_ptr<Resource>);
void F(std::shared_ptr<Resource>&);
```

Общие рекомендации: 

- если не может быть нулевым, то передавать по ссылке
- если может быть нулевым, передавать по значению
- если нужно передать владение - то `std::unique_ptr<>`
- если может быть несколько ссылок на объект, то `std::shared_ptr<>`
- `std::shared_ptr<>` более дорогой, т.к. работает со счётчиком копий

## Использование designated initializers

В C++20 появились _designated initializers_ (именованные инициализаторы) — синтаксическая конструкция, позволяющая явно указывать, какие поля структуры/класса инициализируются. Это делает код более читаемым и защищает от ошибок при изменении порядка полей.

Базовый пример:

```cpp
#include <iostream>

struct Point {
    int x;
    int y;
    double z;
};

int main() {
    // Инициализация с указанием полей
    Point p = {
        .x = 10,
        .y = 20,
        .z = 3.14
    };

    std::cout << "x: " << p.x << ", y: " << p.y << ", z: " << p.z << std::endl;
    return 0;
}
```

Важные особенности:

- Порядок не важен — можно указывать поля в любом порядке
- Необязательная инициализация всех полей — незаданные поля получают значение по умолчанию
- Вложенные структуры
- Массивы структур
- С классами (при соблюдении условий)

Пример инициализации вложенных структур:

```cpp
struct Color {
    int r, g, b;
};

struct Pixel {
    Point pos;
    Color color;
};

Pixel pixel = {
    .pos = {.x = 100, .y = 200},
    .color = {.r = 255, .g = 0, .b = 0}
};
```

Пример инициализации массива структур:

```cpp
Point points[] = {
    {.x = 1, .y = 2, .z = 0.5},
    {.x = 3, .y = 4, .z = 1.5},
    {.x = 5, .y = 6} // z=0.0 по умолчанию
};
```

Пример инициализации класса:

```cpp
class Rectangle {
public:
    int width;
    int height;
    std::string label;
};

Rectangle rect = {
    .width = 100,
    .height = 50,
    .label = "Main window"
};
```

Ограничения:

- Работает только с агрегатными типами (структуры/классы без пользовательских конструкторов, виртуальных функций, приватных/защищённых полей)
- Поля должны быть публичными
- Нельзя использовать с классами, имеющими пользовательский конструктор

Designated initializers особенно полезны:

- При инициализации конфигурационных структур
- В API, где важно явно указать назначение каждого параметра
- При работе с большими структурами, где легко ошибиться с порядком полей
